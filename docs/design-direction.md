# Design Direction

## Product Name

K-RiskHub.

## Brand Rules

- Do not display Cast branding.
- Do not mention Kynovia.
- Use a neutral product identity inspired by the preferred visual direction.

## Interface Principles

- Modern SaaS, minimal and operational.
- Left sidebar navigation.
- Central content focused only on the selected module.
- Top context selector for client/project/portfolio.
- No marketing-style hero pages inside the app.
- Use tables, compact KPI cards, drawers, filters, and clear contextual actions.
- Risk colors should appear mainly in badges, matrix cells, and severity indicators.

## Core Navigation

- Dashboard
- Clientes
- Projetos
- Riscos
- Portfolio
- Relatorios
- Exportacoes
- Administracao

## Login

The login screen should remain classic and quiet:

- Product name
- Microsoft login
- Google login
- Email/password
- Password reset link
- No client or consulting-company branding
